## 怎么读论文

<CardGrid>
  <Card title="先读出问题">
    只看标题、摘要、引言、图表和结论，先答清“解决什么问题、方法叫什么、和已有做法差在哪”。
  </Card>
  <Card title="抓方法和证据">
    读方法、关键图、实验设置和主要结果，讲清主张和证据链，先不纠结附录。
  </Card>
  <Card title="回到代码和失败模式">
    再看 prompt、接口、parser、guardrails，带着“这个设计减少了哪种错误”去读。
  </Card>
  <Card title="复习输出">
    每篇只留三样：一句话贡献、一个关键机制、一个失败模式或边界。
  </Card>
</CardGrid>
