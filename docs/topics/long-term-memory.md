# 长期记忆

长期记忆不是模型内部真的记住了什么，而是外部持久化系统。Agent 在任务开始前检索少量相关记忆，把它们放进本轮上下文；任务结束后，再把值得复用的信息整理写回。

## 三个层次

| 层次 | 含义 |
| --- | --- |
| 短期上下文 | 当前 prompt、当前对话历史、一次 run 里的临时状态。 |
| 会话记忆 | 可跨多轮对话保存的 session history。 |
| 长期记忆 | 跨会话仍然有价值的用户偏好、项目背景、历史决策和踩坑经验。 |

Compact 和长期记忆不是一回事。Compact 是把当前长线程压缩成摘要，帮助继续当前任务；长期记忆是为了未来任务能够继承稳定背景。

## 读写循环

一个可用的 memory 系统至少有五步：

1. 判断当前任务是否需要历史背景。
2. 从 memory store 检索少量相关片段。
3. 把片段注入 LLM 上下文。
4. Agent 执行任务。
5. 任务结束后抽取可复用信息，去重并写回。

关键不是“存得越多越好”，而是检索和注入要克制。把所有历史都塞进上下文，会制造噪音，也会让旧信息污染当前判断。

## 和 agent loop 的关系

长期记忆提供的是任务开始前的背景和任务结束后的沉淀。它不替代 ReAct 里的 Observation，也不替代 SWE-agent 的环境反馈。

可以这样理解：

```text
memory 提供背景
observation 提供当前事实
agent loop 负责在两者之间做决策
```
